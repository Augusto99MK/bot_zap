photoCache

Members

(inner, constant) CACHE_EXPIRATION_TIME :number

Description:

  • In the photoCache.js module, cached photo URLs are stored together with the timestamp of the last update. This timestamp is compared with the current time whenever a photo URL is requested. If the difference between the current time and the last updated time is less than the expiration time (24 hours), the cached URL is used. Otherwise, a new URL is generated and the cache is updated. It was created to try to minimize calls to the api.
